# Tensions

> Living friction log. Append observations during work. Don't solve them here — just capture the signal.
> Format: `- {date}: [{area}] {observation}`

---

- 2026-03-23: [tooling] Biome writes JSON files with tabs but the `write` tool outputs spaces — every new file needs a `bun run format` pass afterward. Minor friction but consistent.
- 2026-03-23: [tooling] Turbo 2.8+ broke backward compat with `packageManager` requirement — t3code reference was slightly stale here.
- 2026-03-23: [tooling] Biome organizeImports has opinions about `node:` builtin ordering that differ from typical convention — need to remember `node:` first when writing imports.
- 2026-03-23: [docs] Pi RPC docs in our PI_INTEGRATION.md show command format with `"command"` field, but actual Pi 0.61.1 uses `"type"` field for commands. Our docs may need correction in Phase 1A.
- 2026-03-23: [tooling] `noUncheckedIndexedAccess` + Biome `noNonNullAssertion` creates friction in test files — can't use `arr[0]` (TS error) or `arr[0]!` (Biome error). Need a `lineAt()` helper pattern.
- 2026-03-23: [tooling] TS project references + `composite: true` conflict with Bun's direct `.ts` exports pattern. Packages export source `.ts` files, but project references expect compiled `.d.ts` in dist/. Solution: don't use project references in app tsconfigs — let Bun's module resolution handle it.
- 2026-03-23: [tooling] Biome `useLiteralKeys` + `organizeImports` add small friction when writing code — bracket notation for env vars and import ordering of `type` vs value imports are unintuitive at first.
- 2026-03-23: [plan] Several items in Phase 1C (1C.10, 1C.12, 1C.14) were already implemented by the time they came up for execution. Plan granularity may need adjustment — or items should be checked off as side-effects of earlier work when they're clearly done.
- 2026-03-23: [deps] react-markdown v10 changed export format (default vs named) — documentation/examples online still show v9 patterns. Version-specific API checks needed.
- 2026-03-23: [tooling] Biome's static analysis for a11y can't track `alt` attribute through prop spreading — had to restructure img component to not spread `...props`.
- 2026-03-23: [architecture] Pi RPC has no `list_sessions` command, which limits session switching UX. Need server-side session directory scanning or Pi protocol extension for proper session management.
- 2026-03-23: [architecture] Server-side session listing reads Pi's internal file structure (`~/.pi/agent/sessions/`). This is a coupling to Pi's implementation details — if Pi changes its session storage format, our listing breaks. Acceptable for now but fragile long-term.
- 2026-03-23: [deps] Electrobun distributes raw `.ts` source files instead of compiled `.d.ts` — this means `skipLibCheck` is ineffective for suppressing type errors in Electrobun code. Had to disable `exactOptionalPropertyTypes` in the desktop package. Third-party TS types leak into our strict checking.
- 2026-03-23: [types] Desktop importing server `.ts` source files means server code is typechecked under desktop's tsconfig (different `lib`, different strictness). DOM lib overrides Bun-specific type augmentations (e.g., `ReadableStream[Symbol.asyncIterator]`). Cross-package `.ts` imports are convenient but create hidden type environment conflicts.
- 2026-03-23: [types] Electrobun's `BrowserWindow.on()` types all event handlers as `(event: unknown) => void` — can't get type safety on event data without casting inside the callback. Minor ergonomic friction that could be improved upstream.
- 2026-03-23: [electrobun] `exitOnLastWindowClosed` default is dangerous for apps with async cleanup — force-exit races with server/process shutdown. Must explicitly disable and manage exit manually.
- 2026-03-23: [architecture] Native menu accelerators overlap with web keyboard shortcuts (Cmd+N, Cmd+B, Cmd+L). In desktop mode, native accelerators intercept before keydown reaches the webview. Need dual handling: native menu action → same operation, web shortcut → same operation. This duplication is necessary but needs careful coordination to avoid double-execution.
- 2026-03-23: [architecture] Menu action forwarding in dev mode is a gap — when PIBUN_DEV_URL is set, pibunServer is null so broadcastPush can't reach the web app. Desktop menus become non-functional for forwarded actions. Users must use keyboard shortcuts. Could fix by having the desktop connect to the external server as a WS client, but that's complexity for a developer-only scenario.
- 2026-03-23: [distribution] Electrobun CLI not in PATH by default — must use node_modules/.bin/electrobun or npx. Root build:desktop script uses `cd apps/desktop && electrobun build` which may fail without npx. Consider adding a bun script wrapper.
- 2026-03-23: [distribution] Plan item 2C.3 says "AppImage" but Electrobun uses self-extracting installer archives. Plan titles slightly stale relative to actual Electrobun capabilities. Same will likely apply to 2C.4 "NSIS installer" — Electrobun may use its own wrapper format for Windows too.
- 2026-03-23: [distribution] No cross-compilation in Electrobun — Linux, Windows, and macOS builds each require their native platform. CI matrix will need runners for all three OSes. Local testing of non-host builds impossible without VMs.
- 2026-03-23: [distribution] Confirmed: 2C.4 "NSIS installer" is also stale — Electrobun uses self-extracting exe, not NSIS. Three consecutive plan items (2C.1 "AppImage", 2C.4 "NSIS") had incorrect installer format names. Original plan was written before deeply studying Electrobun's build system.
- 2026-03-23: [architecture] Server `handleWsMessage` function scope required threading `hooks` as an explicit parameter — the `config` closure variable was not accessible inside the WebSocket message handler. This pattern (threading context through dispatch layers) accumulates — now rpcManager, connections, sendPush, AND hooks all pass through. Consider consolidating into a single context object passed once.
- 2026-03-23: [architecture] Item 1.1 says "Extend PiRpcManager" but PiRpcManager already supported multiple sessions. The real work was in the WS binding layer (WsConnectionData, HandlerContext, event tagging). Plan item description slightly misleading — the change was server-wide, not isolated to PiRpcManager.
- 2026-03-23: [architecture] Tab creation during session start requires inline helpers (not imported) to avoid circular dependency between tabActions ↔ sessionActions. The pattern of "coordinator module that imports both" doesn't work when both modules need each other. Solved by keeping tab creation simple enough to inline.
- 2026-03-23: [deps] `node-pty` N-API prebuild loads in Bun but `posix_spawnp` fails at runtime (ENXIO on FD read). Bun's subprocess FD handling differs from Node's. Had to use `bun-pty` (Rust FFI) instead. Bun's N-API compat layer is incomplete for native addons that manipulate file descriptors.
- 2026-03-23: [platform] Electrobun lacks native "Save As" file dialog — only `openFileDialog` (open mode). For export save, used folder picker + `Bun.write()` workaround. Users pick a folder, file is written with auto-generated name. Not ideal UX but functional.
- 2026-03-24: [theming] `text-text-primary` naming is verbose (Tailwind prefix + token prefix both say "text"). Unavoidable with Tailwind v4's @theme naming convention where `--color-{name}` maps to `{utility}-{name}`. Consider shorter token names (e.g., "primary" instead of "text-primary") if this becomes a readability issue.
- 2026-03-24: [docs] AGENTS.md and CAPABILITY-MAP.md in `.pi/` are auto-loaded by Pi agent harness, but SOUL.md and HUMAN.md in `.agents/` are only loaded because AGENTS.md tells the agent to read them. After consolidation into CLAUDE.md, the `.pi/AGENTS.md` file still auto-loads — need to either delete it or replace with a redirect to CLAUDE.md. Deletion is cleaner.
- 2026-03-24: [lint] Biome `useSemanticElements` rule wants `<dialog>` instead of `role="dialog"` on divs. But native `<dialog>` has different positioning/backdrop behavior that doesn't match our overlay pattern (click backdrop to close). The `biome-ignore` comment only works when `role` is on the same line as the opening tag — multi-line JSX attributes break the suppression. Workaround: remove `role` entirely from simple overlays.
- 2026-03-24: [architecture] Markdown `components` override was a module-level constant, which prevents any dynamic behavior (e.g., image click handlers that need React state). Had to refactor to factory function + `useMemo`. This means every MarkdownContent instance creates its own components object reference — might cause react-markdown to re-render its tree. Monitor for perf impact in long conversations.
- 2026-03-24: [docs] Pi rpc.md docs show `PiSlashCommand` with `location`/`path` fields, but actual Pi rpc-types.ts uses `sourceInfo: SourceInfo`. Docs are outdated relative to the code. Always trust pi-mono source over docs when there's a mismatch.
